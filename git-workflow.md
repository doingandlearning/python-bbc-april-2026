# Getting Started with Git (Quick Guide)

You don’t need to master Git today.  
Focus on understanding the basics and trying it out yourself.

---

## What is Git?

Git is a **version control system** — think of it as a **time machine for your code**.

It lets you:
- Save snapshots of your work (**commits**)
- Go back to previous versions
- Work on different ideas safely (**branches**)
- Share your code with others (**GitHub, GitLab, etc.**)

---

## The Basic Workflow

Most beginners only need this:

1. Edit your files  
2. Stage changes  
3. Commit a snapshot  
4. Push to a remote repository  

```bash
git add .
git commit -m "Your message"
git push
```

## Key Commands (First 3 Months)

Focus on these:

```bash
git init     # start a repo
git add      # stage changes
git commit   # save snapshot
git push     # send to remote
git pull     # get latest changes
```

---

## How to Actually Learn Git

Watching videos is not enough.

To learn Git properly:

1. Create a project
2. Make changes
3. Commit often
4. Break something
5. Try to fix it

That’s where the real understanding comes from.

You don’t need to understand everything.
